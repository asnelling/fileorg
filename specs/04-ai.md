# Spec: AI Categorization Module

## Purpose

Uses a locally-running Ollama model to assign a hierarchical category to each file based on the clues collected by plugins. The AI module is intentionally thin — it converts clues to a text prompt, calls Ollama, and parses the JSON response.

## Files

### `src/fileorg/ai/prompts.py`

Defines the system prompt and user prompt builder.

#### System Prompt (constant string `SYSTEM_PROMPT`)

```
You are a file categorization assistant. You receive clues about a single file
and assign it to the most specific applicable category using a slash-delimited
hierarchy (e.g. "Photography/Travel/Europe" or "Documents/Legal/Contracts").

Rules:
- Return ONLY valid JSON. No markdown fences, no prose, no explanation outside JSON.
- Use exactly this schema:
  {"category": "<slash/delimited/path>", "confidence": <0.0-1.0>, "reasoning": "<one sentence>"}
- Prefer specificity: "Documents/Legal/Contracts" over "Documents".
- Use title case for category components (e.g. "Photography" not "photography").
- If clues are insufficient, return category "Uncategorized" with confidence <= 0.2.
- Do not invent facts not present in the clues.
- confidence reflects how certain you are given the available evidence.
```

### `src/fileorg/ai/client.py`

Wraps the `ollama` Python library.

```python
import json
import ollama as _ollama
from fileorg.ai.prompts import SYSTEM_PROMPT

_FALLBACK = {"category": "Uncategorized", "confidence": 0.0, "reasoning": ""}

class OllamaClient:
    def __init__(self, model: str = "llama3.2", timeout: int = 120) -> None:
        self.model = model
        self.timeout = timeout

    def is_available(self) -> bool:
        """Returns True if Ollama is running and the model is available."""
        try:
            models = _ollama.list()
            return any(m['name'].startswith(self.model) for m in models.get('models', []))
        except Exception:
            return False

    def categorize(self, user_prompt: str) -> dict:
        """
        Calls ollama.chat() with structured JSON output mode.
        Returns {'category': str, 'confidence': float, 'reasoning': str}.
        On any error or parse failure, returns _FALLBACK.
        """
        try:
            response = _ollama.chat(
                model=self.model,
                messages=[
                    {"role": "system", "content": SYSTEM_PROMPT},
                    {"role": "user", "content": user_prompt},
                ],
                format="json",
                options={"num_predict": 200},  # cap tokens for speed
            )
            raw = response["message"]["content"]
            parsed = json.loads(raw)
            return {
                "category": str(parsed.get("category", "Uncategorized")),
                "confidence": float(parsed.get("confidence", 0.0)),
                "reasoning": str(parsed.get("reasoning", "")),
            }
        except Exception:
            return dict(_FALLBACK)
```

## Behavior Details

### Model selection
- Default: `llama3.2` (3B parameter model; fast on CPU, ~2GB RAM)
- Alternative: `mistral` (7B, better quality, needs ~4GB RAM)
- Configured via `--model` CLI flag; passed through to `OllamaClient`

### `format="json"` in Ollama
- Instructs the model to output valid JSON only
- Supported by llama3.2, mistral, and most modern models
- Not 100% guaranteed — `json.loads()` failure is handled by returning `_FALLBACK`

### Prompt length management
- User prompt is capped at ~2000 characters total
- If clues would exceed this, `build_user_prompt` truncates:
  - Limit `detected_text` OCR clue to 300 chars
  - Limit total clue list to 30 entries (most informative first: EXIF > filename > OCR > clueless)

### Timeout
- `timeout=120` (2 minutes) per file; configurable
- Ollama does not natively support per-call timeouts in the Python client; wrap call in `concurrent.futures.ThreadPoolExecutor` with `.result(timeout=timeout)` to enforce

### Offline fallback
- If `is_available()` returns `False` at scan start: warn user via Rich console
- If `--dry-run` flag set: skip AI stage entirely
- Files left in `clued` status (not `categorized`) when AI unavailable — they will be picked up on resume when Ollama is available
